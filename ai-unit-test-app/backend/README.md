# AI Unit Test Agent - Backend API

> **This is a sub-project of the [AI Unit Test Agent Platform](../README.md). Please check the main README for overall setup and architecture details.**

This is the backend server for the **AI Agent for Automated Unit Test Generation and Optimization in C#**. It handles requests from the frontend, manages temporary C# file generation, clones/parses GitHub repositories using Roslyn Parser CLI, creates GitHub Pull Requests, and runs the core Python AI runner scripts to perform unit test generation, evaluation, and optimization.

## Features

- **Model Listing**: Returns supported AI models (GPT-4o Mini, Llama 3.3 70B, DeepSeek-R1, DeepSeek-V3).
- **Automated Test Generation**: Runs workflows to generate and refine unit tests using C# source code.
- **GitHub Ingestion**: Clones public GitHub repositories, parses class structures, and creates remote Pull Requests automatically.
- **Segregated Dashboard Feeds**: Exposes isolated endpoints to query playground submissions, GitHub ingest runs, and CI/CD pre-commit hooks logs.
- **Interactive API Docs**: Built-in Swagger/OpenAPI documentation.

## Endpoints Summary

### `GET /api/models`
Returns list of supported AI models.

### `POST /api/generate`
Executes unit test generation.
- **Request Body (`GenerateDto`)**:
  - `sourceCode` (string): The C# source code to test.
  - `model` (string): The AI model identifier (e.g., `gptmini`).
  - `workflow` (string): The execution workflow (e.g., `agent`).
  - `source` (string, optional): `'playground'` or `'github'`.
  - `filePath` (string, optional): Target file path context.
- **Response**: The output logs and generated tests from the AI runner scripts.
- **Side Effect**: Saves a copy of the run output, timestamp, and source code under `ai-unit-test-benchmark/benchmark_runner/results/web_demo/` or `results/github_demo/` (depending on the `source` payload) to isolate user demo runs from the benchmark dataset.

### `POST /api/github/clone`
Clones a public GitHub repository locally, parses C# source files using the Roslyn Parser CLI, and extracts public method details.
- **Note**: This endpoint automatically cleans up old `temp_repo_*` directories before cloning to prevent disk space exhaustion.
- **Request Body**: `{ "repoUrl": string }`
- **Response**: `{ "tempDir": string, "files": Array<{ filePath: string, className: string, methods: Array<{ methodName: string, signature: string, body: string }> }> }`

### `POST /api/github/create-pr`
Creates a Git branch, commits the generated test, pushes it to the remote repository, and opens a GitHub Pull Request using `GITHUB_PAT`.
- **Request Body**: `{ "repoUrl": string, "tempDir": string, "filePath": string, "testCode": string, "githubPat"?: string }`
- **Note**: `githubPat` can be supplied directly from the UI. If omitted, the server falls back to `GITHUB_PAT` in `benchmark_runner/.env`.
- **Response**: Detailed Pull Request object returned by the GitHub API.

### `POST /api/cicd/webhook`
Webhook endpoint for automated GitHub Action CI/CD pipelines. This triggers the native execution workflow without UI interaction.
- **Request Body**: `{ "repoUrl": string, "prNumber": string, "branch": string, "workflow"?: string }`
- **Default Workflow**: `compiler-guided-multi-agent`
- **Response**: `{ "status": "Pipeline triggered", "repoUrl": string, "workflow": string }` (Runs asynchronously in the background)

### `GET /api/dashboard/benchmarks`
Recursively scans the `results/reports` and `benchmark_datasets` directories to build and return a complete array of all gold standard benchmark dataset runs with matched source code and expected tests.

### `GET /api/dashboard/static-summary`
Scans and returns pre-calculated static summary report JSON files (Overall, Category, Cost Efficiency with Worker/Evaluator breakdown, Failure, Self-healing, Latency, Evaluator-Guided, and Candidate Selector summaries) generated by the Python report compiler for a given dataset version and workflow.
- **Note**: The returned static summaries now include separated `conditional` (success-only) and `effective` (overall) evaluation metrics (e.g., line coverage, branch coverage, mutation score, evaluator score).
- **Query Parameters**:
  - `version` (string): The dataset version (e.g. `v1`).
  - `workflow` (string): The workflow name (e.g. `agent`).

### `GET /api/dashboard/web-demo`
Scans and returns all user-triggered playground submissions stored under `results/web_demo/`, sorted by timestamp descending.

### `GET /api/dashboard/github-demo`
Scans and returns all GitHub Ingestion submissions stored under `results/github_demo/`, sorted by timestamp descending.

### `GET /api/dashboard/ci-cd`
Scans and returns all Git pre-commit hook runs stored under `results/ci_cd/`, sorted by timestamp descending.
